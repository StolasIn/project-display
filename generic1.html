Data structure

  1. stack
  2. queue
  3. linked_list
  4. tree

  1. stack :


    • 只有一個開口，只能從該開口插入、刪除東西的資料結構
    • stack是一種LIFO的資料結構，函數呼叫疊代或網頁上下跳頁的時候就是一種基於stack的表現
    • stack可以使用linked_list或array實作，且需要一個變數紀錄最頂端的元素
    • 以array實作時非常直觀，就是單純把陣列尾部當作開口，但無法動態配置記憶體，在宣告時就決定了最大空間上限
    • 以linked_list實作時可以在執行時期動態配置記憶體，但需要比陣列多使用一個pointer空間
    • 在c++中有提供stack的資料結構，宣告方法為<stack> stack1;

  2. queue :


    • 有兩個開口，一個負責插入資料，一個負責刪除資料的資料結構
    • queue是一種FIFO的資料結構，有點類似生活中的排隊
    • stack可以使用linked_list或環形array實作，需要兩個變數分別紀錄最頭的元素與最尾巴的元素
    • 以環形陣列實作時需要保留一格空間作比對，才不會發生覆蓋資料的問題
    • 在c++中有提供queue的資料結構，宣告方法為<queue> queue1;

  3. linked_list :


    • linked_list的每個節點通常由兩個元素(data與指向下一個節點的指標)組成(double linked_list有三個)
    • linked_list的插入與刪除都是O(1) (不考慮尋找點的過程)
    • linked_list不像array一樣能random access memory，所以linked_list的traverse需要 O(n)
    • 比起array更適合用在頻繁做插入刪除且個數不多的資料，因為當在陣列裡頻繁的做插入刪除時需要整組陣列的移動

  4. tree :


    • 樹會有一個根節點
    • 樹的父節點與子節點會有parent child的關係
    • level被稱為樹高，是從根結點到最遠葉節點的距離
    • 建議參考 : 樹的簡介與類型介紹
